## Summary

AI 代理正在成为企业软件的主要用户，而非人类。这一转变催生了“无头”软件架构——用户界面被剥离，软件功能通过 API 直接暴露给 AI 代理调用。传统 SaaS 产品正从“交互系统”退化为“记录系统”，企业软件的价值链和竞争壁垒正在被重塑。

## 核心观点

- **“无头”软件兴起**：当 AI 代理成为主要用户时，图形界面不再是必需。软件的核心变为可通过 API 编排的功能模块，代理直接调用这些模块完成任务。
- **SaaS 角色转变**：Salesforce 等传统 SaaS 正从“交互系统”转向“记录系统”。用户不再直接操作界面，而是通过代理与底层数据和逻辑交互。Salesforce 的 Headless 360 公告正是这一趋势的体现。
- **企业软件粘性来源**：替换 SAP、Salesforce 等系统的难度远超表面所见。其粘性不仅来自功能，更来自深度嵌入企业流程的数据模型、合规逻辑和集成网络。
- **初创公司机会**：AI 重塑软件栈的过程中，最大的机会在于构建代理原生工具、编排层和新的数据交互方式，而非直接替换现有巨头。

## 值得展开的细节

- **API 与代理工作流**：代理需要的是可组合的 API，而非完整的应用界面。这要求软件架构从“以人为中心”转向“以代理为中心”，MCP（模型上下文协议）等标准正在推动这一变化。
- **企业架构演变**：传统三层架构（展示层、逻辑层、数据层）被打破。展示层被代理取代，逻辑层被 AI 编排重构，数据层的访问模式从“人查”变为“代理调”。
- **Salesforce Headless 360**：这一产品将 Salesforce 的核心能力（数据模型、自动化规则、分析）通过 API 暴露，允许代理直接操作客户数据，而无需通过标准界面。这是传统 SaaS 巨头应对代理趋势的典型案例。

## 对 AI、投资、产品和研究的启示

- **AI 方向**：代理需要更结构化的 API 和语义清晰的数据模型。当前多数企业 API 是为人类开发者设计的，需重构为“代理友好”格式。
- **投资视角**：投资“记录系统”类 SaaS 需重新评估其长期价值。代理层和编排层的初创公司可能捕获更大价值，但需解决企业集成和信任问题。
- **产品策略**：现有 SaaS 产品应尽早暴露核心功能为 API，并设计代理交互模式。忽视这一趋势可能导致产品被“无头化”替代。
- **研究课题**：代理间的协调协议、企业数据访问权限在代理场景下的重构、以及“无头”架构下的安全与合规模型，都是待解决的关键问题。
