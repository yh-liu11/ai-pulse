## Summary

AI 基础设施正从“开发者体验”转向“智能体体验”。传统云架构（如 Kubernetes）为人类设计，依赖文档和仪表盘，但智能体无法自行填补缺失的上下文。因此，基础设施必须提供更紧凑、可编程的接口，让智能体能够快速编写代码、运行、检查输出、调试并重试，实现高速迭代。

**核心要点**
- **智能体体验取代开发者体验**：Modal 已将 SDK 团队的重心转向智能体体验，因为将基础设施需求以装饰器形式与代码共存，比让智能体阅读数百个 YAML 文件更高效。
- **弹性推理与 GPU 快照**：核心用例是为自定义模型提供弹性推理，通过 GPU 状态快照大幅缩短冷启动时间，应对突发流量。
- **沙箱是智能体的关键原语**：智能体需要隔离环境来执行代码。强化学习（RL）的 rollout 阶段可能需要同时启动 10 万个沙箱，这对基础设施的弹性要求极高。
- **可观测性比代码更重要**：当智能体编写代码时，人类不再直接审查代码，而是依赖强大的仪表盘和 CLI 工具来理解系统行为并做出决策。

**值得展开的细节**
- **DeFlash 与推测解码**：Modal 开源了基于块的推测解码技术 DeFlash，通过小模型预测、大模型批量验证，在内存带宽受限的场景下显著提升推理性能。
- **17 云超级云策略**：Modal 不拥有数据中心，而是在 17 个云提供商之上构建统一的容量池和可靠性层，专注于软件差异化。
- **网络化沙箱与 RDMA**：支持沙箱间通过私有 IPv6 网络通信，并为多节点分布式训练提供 RDMA 网络，加速内存和 KV 缓存迁移。

**对 AI、投资、产品及研究的启示**
- **产品设计**：为智能体设计的产品应优先考虑可编程 API 和 CLI 可访问性，而非图形界面。Modal 甚至通过观察智能体的“幻觉”需求来发现产品功能。
- **投资视角**：智能体云是明确的未来方向。能够提供从数据准备、训练到推理，以及后台智能体部署全生命周期的平台，具有巨大价值。
- **技术架构**：AI 工作负载（尤其是智能体）导致 GPU 与 CPU 的消耗比例从 8:1 趋向 1:1，要求计算资源必须紧密协同定位，这对底层网络和调度提出了新挑战。
