## Summary

## 简要总结

本期内容包含两个独立部分。第一部分是 Claire 对 Anthropic 新模型 Sonnet 5 的系统性评测。她没有依赖感觉，而是构建了一个可重复的基准测试，将 Sonnet 5 与 Sonnet 4.6、Opus 4.8、GPT-5.5 和 Gemini 3 Pro 进行盲测，任务涵盖 PRD 撰写、原型构建、智能体任务和智能体个性。结果显示，Sonnet 5 的性价比定位尴尬，其质量在个人偏好排名中垫底，且自动化评分与人类判断严重背离。第二部分是 Alessio Fanelli 分享如何用手机管理自主编程智能体。核心转变是从“智能体提示者”变为“智能体管理者”，利用 OpenAI Symphony、Linear 和云 VPS 实现异步工作。他还展示了用 AI 在 eBay 上寻找低价宝可梦卡牌的套利案例，揭示了 AI 如何压缩传统的信息优势。

## Core takeaways

1.  **Sonnet 5 的定价与性能不匹配。** 其定价（每百万输入 token 2 美元，输出 token 10 美元）介于之前的 Sonnet 和 Opus 模型之间，但在 Claire 的基准测试中，其个人偏好排名接近垫底。成本优势只有在质量也满足特定用例时才成立。
2.  **可重复的基准测试比一次性感受检查更有价值。** Claire 构建的“How I AI Bench”使用冻结的输入、固定的评分标准和相同的任务，使得跨模型、跨时间的比较成为可能。这是评估模型迭代真实进步的关键。
3.  **自动化评分（LLM-as-judge）不可靠。** GPT-5.5 和 Opus 4.8 作为评判时，评分过于集中在中段，且无法发现人类一眼就能看出的问题，如损坏的原型或忽略的线框约束。Claire 的个人排名与自动化排名几乎完全相反。
4.  **智能体个性是选择模型的关键因素。** Sonnet 4.6 在基准测试中并非最高分，但因其对话风格和个性，仍是 Claire 日常智能体工作的首选。这强调了非量化指标在开发者体验中的重要性。
5.  **从“智能体提示者”到“智能体管理者”的转变是释放生产力的关键。** Alessio 的经验表明，将智能体运行环境迁移到云 VPS，并通过 Linear 等工具进行异步任务管理，能消除本地运行的摩擦，实现真正的异步工作流。
6.  **Token 成本追踪是智能体系统的基础设施。** 记录每个任务的 token 消耗（Alessio 的任务消耗从 1500 万到 2.21 亿 token 不等）是优化规格说明和工具链的必要反馈循环，但多数设置都缺少这一环节。
7.  **AI 正在压缩传统的信息优势。** 通过 Codex 和浏览器访问权限，一个精心编写的提示词就能完成过去需要大量人力、时间和专业知识才能完成的套利任务，如寻找低价高价值宝可梦卡牌。

## Details worth expanding

-   **基准测试的构建方法：** Claire 使用 Claude Code 读取历史会话记录，生成了与其实际工作相关的评估任务。她手动为 5 个模型的 64 次生成打分（1-5 分），并添加注释。她认为，构建一个用于评分和导出 JSON 的 HTML 页面只需约 45 分钟，但价值巨大。
-   **加权指数的创建：** 当自动化评分与个人判断背离时，Claire 创建了一个 70% 个人权重、30% LLM 权重的混合指数。这使 Sonnet 4.6 跃升至第一，揭示了评分标准需要更好地编码个人偏好。
-   **任务特定推荐：** 基于加权指数，Claire 给出了具体建议：GPT-5.5 用于 PRD，Sonnet 4.6 用于原型和闲聊，Opus 4.8 或 Sonnet 5 用于代码库导航。Opus 4.8 在复杂、密集的 UI 工作上仍有价格溢价的价值。
-   **Symphony 的本质：** Claire 明确指出，OpenAI Symphony 并非魔法，而是一个高度规范化的 Markdown 规范文件，告诉模型如何行为。当前模型已经足够强大，能忠实地遵循这个规范。
-   **技能文件的维护：** Alessio 建议每隔几个月清理一次智能体的技能文件。模型倾向于添加而非替换指令，导致文件随时间增长而自相矛盾。文件应保持简短、紧凑，并明确说明智能体需要询问什么。
-   **智能体的感知能力：** Kernel Labs 构建了 Glimpse（一个 Playwright 扩展），因为智能体运行的瓶颈不是编排，而是遇到 UI 歧义时请求帮助。提供截图、视觉差异等更好的感知工具，能显著延长自主运行时间。
-   **异构数据的商业机会：** Alessio 指出，AI 最大的未开发机会在于基于异构数据的业务，如交易卡、古着、鱼类库存等。这些数据不一致、视觉化且依赖上下文，LLM 是首个无需大量预处理就能处理这种混乱的技术。
-   **小型企业的非对称优势：** Alessio 观察到，AI 为一人或两三人规模的小型企业带来的杠杆效应是非对称的，这是大型组织无法以相同成本复制的。这是一个被低估的 AI 叙事。

## Implications for AI, investing, products, or research

-   **对 AI 模型评估的启示：** 当前主流的自动化基准测试和 LLM-as-judge 方法存在严重缺陷，无法捕捉对实际使用者至关重要的质量和体验维度。未来的评估体系必须系统性地整合人类偏好和领域专家的隐性知识。
-   **对投资者的启示：** 投资决策不应仅依赖公开的模型基准分数。需要深入理解特定用户画像（如开发者）的“品味”和“个性”偏好，这些因素可能比原始能力更能驱动实际采用和付费意愿。同时，关注为小型企业提供非对称 AI 杠杆的工具和平台。
-   **对产品设计的启示：** 智能体产品的设计重心应从“如何编写提示词”转向“如何高效管理异步任务”。关键功能包括：清晰的成本追踪仪表盘、健壮的状态机（如 Linear 集成）、多通道通信，以及为智能体提供更丰富感知（视觉、上下文）的工具。
-   **对研究的启示：** 需要研究如何将难以量化的“品味”和“个性”编码进评估标准。同时，探索如何构建能自我维护、避免指令膨胀的智能体技能系统，以及如何让模型更好地利用视觉信息解决 UI 歧义，是提升自主智能体实用性的关键方向。
