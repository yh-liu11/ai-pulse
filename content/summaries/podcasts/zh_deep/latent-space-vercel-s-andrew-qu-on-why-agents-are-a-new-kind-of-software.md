## Summary

Vercel 首席软件官 Andrew Qu 认为，智能体（Agent）是一种全新的软件形态。它们不像传统 Web 应用那样可预测，其交互、界面和输出都更加动态，因此需要不同的构建原语，例如上下文管理、工具调用、可恢复性和长时间运行支持。Vercel 从构建自己的编码智能体 v0 的过程中积累了经验，将这些解决方案提炼为可复用的库，并最终整合成智能体框架 eve。Vercel 自身也正在转变为一个智能体，在其网站、Slack 和仪表盘中部署智能体，代表用户执行操作。

## 核心要点

1.  **智能体是新型软件**：智能体不是传统应用的简单延伸。其非确定性、动态交互和长时间运行特性，要求全新的构建范式。基础设施可能相似，但构建方式、所需原语完全不同。
2.  **框架源于内部实践**：Vercel 的智能体框架 eve 并非凭空设计。它源于在构建 v0 产品时遇到的真实痛点，如模型切换、回退机制、运行可恢复性等。团队将解决方案积累为可复用原语，最终形成规范化的框架。
3.  **技能是“前向纠错”机制**：由于模型知识可能过时（例如推荐已弃用的产品），技能（Skills）作为便携式、按需加载的知识，可以引导智能体使用最新信息。企业应发布最新产品版本的技能，并审计、更新旧内容。
4.  **网络正变得“智能体可读”**：随着机器人流量上升和人类流量停滞，网站的未来是让智能体和机器人尽可能容易地访问。Vercel 已开始检测智能体请求，并直接提供 Markdown 格式内容，而非需要解析的 HTML。
5.  **人机回环取决于任务**：未来并非全自主或全人工干预。对于定义明确、输出可预期的任务，可让智能体自主循环完成；对于需要精细控制的工作，则应保持人工介入和引导。

## 值得展开的细节

### 从 Web 应用到智能体的演进
Andrew Qu 的角色是“首席软件官”，负责通过任何形式交付影响力，包括内部工程、外部实验和前沿探索。他构建了 MCP 库和 skills.sh 平台。Vercel 的起源是简化 Web 应用部署，但近期观察到开发者从构建页面转向构建智能体。这一转变催生了对新工具的需求，最终导致了 eve 框架的诞生。

### 为什么需要专用框架
Qu 在 Vercel 内部推动“每个办公桌上都有一个智能体”的目标时，构建了一个成功的数据智能体。在此过程中，他总结出文件系统智能体、技能、压缩和子智能体等一系列最佳实践。eve 框架的初衷，就是将这些探索成果规范化，让其他开发者不必重复同样的试错过程。

### 智能体的适用场景
智能体特别适合需要一定推理能力的重复性任务。Vercel 内部用例包括：法律合同红线标注的初稿、营销回顾与联系人识别、针对数据存储编写查询。这些任务不是固定自动化流程，因为系统需要解读情况并决定如何行动。

### 技能的重要性
技能被视为便携式、按需加载的知识。Qu 举例说明，模型有时仍会推荐 Vercel 已弃用多年的 Postgres 产品。通过技能，可以告知智能体该产品已弃用，并引导其使用当前方案。这是一种在内容全面审计更新之前，“前向纠正”模型行为的有效方法。他建议企业发布最新产品的技能，同时审计现有内容，识别并更新过时信息。

### 智能体可读的网络
Vercel 的报告显示，机器人流量上升，人类流量停滞或下降，但展示量增加，因为智能体和机器人在更频繁地访问网站。因此，网站的未来是尽可能对机器人和智能体友好。Vercel 已实施检测机制：当识别到智能体请求时，直接提供 Markdown 格式内容，使其更易读取，而非强迫其处理为视觉浏览器设计的 HTML。这意味着未来可能为人类和智能体提供两种不同的体验：人类继续接收视觉网站，智能体则获得结构化、机器可读的表示。

### 下一步方向
Qu 当前最感兴趣的领域是**多人智能体开发**。团队协作时，成员之间难以共享上下文。例如，某人可能掌握让前端界面一次成功的技巧，而其他人不知道。他关注如何在队友间共享上下文，并允许他们共同贡献。

### Vercel 的平台化与智能体化
关于智能体是独立应用类别还是软件标配能力，Qu 认为取决于构建者。对 Vercel 而言，平台本身正在成为智能体。他们在网站、Slack 和仪表盘中部署了能代表用户执行操作的智能体。Vercel 的目标是让整个平台对智能体友好，并在很多方面使平台自身成为一个智能体。在平台构建上，Vercel 重视提供智能体生命周期特定部分的合作伙伴，但也希望开发者能极简起步。若将 eve 部署到 Vercel，可开箱获得可观测性和评估功能。他们希望让体验更全面，同时易于集成合作伙伴，而非拥有每个组件。

## 对 AI、投资、产品或研究的启示

*   **AI 基础设施的新原语**：智能体作为新软件形态，催生了对上下文管理、工具调用、运行可恢复性、安全代码执行沙箱等新原语的需求。这为基础设施和工具链创业公司提供了明确的产品方向。
*   **“智能体可读”成为网络新标准**：网站需要为智能体提供结构化、机器可读的内容（如 Markdown）。这可能会催生新的 SEO 实践（面向智能体优化）、内容交付标准和工具。忽视这一趋势的企业将在智能体经济中变得不可见。
*   **技能市场与知识校正层**：技能作为模型知识的“前向纠错”机制，可能发展成一个新的分发渠道和市场。企业需要管理其产品和 API 的“技能版本”，确保智能体能准确使用其服务。这为开发者工具和开发者关系带来了新的工作维度。
*   **投资视角**：投资于智能体框架、沙箱环境、智能体可观测性、评估工具以及“智能体可读”内容基础设施的公司，可能具备长期价值。Vercel 的策略表明，从应用层（v0）向下提炼框架（eve）并开放，是一条可行的路径。
*   **产品设计**：产品经理和工程师需要重新思考用户界面。未来的产品可能同时拥有面向人类的视觉界面和面向智能体的机器接口。人机回环的设计不应是二元的，而应根据任务的风险和明确性，设计灵活的反馈周期。
