## Summary

### 简要总结
Lilian Weng 发布了一篇关于“Harness Engineering”（框架工程）的新综述，系统梳理了 35 篇相关论文。核心观点是，AI 的递归自我改进（RSI）未来将高度依赖外部框架，而非模型直接修改自身权重。她认为，即使许多框架功能最终会被内化到模型核心，但“指定目标和上下文的需求永远不会消失”。这篇综述不仅总结了当前已验证的框架设计趋势和优化文献，也为其联合创办的 Thinky 公司的技术方向提供了线索。

### Core takeaways
1.  **框架是 RSI 的核心**：Lilian Weng 将递归自我改进（RSI）的讨论焦点从模型内部权重的自我修改，转移到了外部“框架”上。框架负责设定目标、提供上下文、调用工具并管理迭代循环。
2.  **目标与上下文不可消除**：一个关键论点是，无论模型本身变得多么智能，向系统明确“要做什么”和“在什么情境下做”的需求是永恒的，这正是框架的核心价值。
3.  **设计趋势已形成**：文章总结了当前已被验证的主流框架设计模式，这些模式是所有构建智能体系统的人都应了解的。
4.  **框架优化文献综述**：文章重点回顾了框架优化的相关研究，从著名的 **ACE 论文**到更近期的 **Meta-Harnesses（元框架）** 趋势，后者指的是能够设计或优化其他框架的框架。
5.  **与 Thinky 的关联**：这篇综述被视为对其联合创办的 Thinky 公司技术思路的暗示，其方向超越了单纯的“交互模型”，可能指向更高级的框架自动化。

### Details worth expanding
- **框架工程成为智能体设计的中心**：Lilian Weng 的文章被广泛引用，作为行业将智能体设计重心转向框架工程的标志。Sakana 的总结将其与《The AI Scientist》、ShinkaEvolve 和 Darwin Gödel Machine 等项目联系起来。LangChain 也呼应了这一转变，推出了新的 **Deep Agents 课程**和一个**开源框架项目**。
- **产品化趋势明显**：
    - **Anthropic** 推出了 **Claude Cowork** 的移动和网页版，将 Claude 定位为在后台运行任务的“队友”，而非单纯的前端聊天界面。
    - **Google** 的 **Gemini API Managed Agents** 增加了后台执行、远程 MCP 服务器、自定义函数调用和凭证刷新等功能，正在将框架能力产品化。
- **框架基础设施日趋完善**：
    - **Codex Mobile iOS** 增加了任务管理、过滤差异、SSH 密钥登录等面向操作者的功能。
    - **Hermes Agent** 增加了可插拔的密钥管理器，并原生集成了 1Password，支持将会话和数据集导出到私有 Hugging Face 仓库。
    - **Weaviate 1.38** 使其 MCP 服务器正式可用，支持在运行时动态开启写入权限，无需重启。
    - 一个实验性模式来自 **@omarsar0**，他使用 Dial MCP 服务器让智能体可以通过电话、短信或 iMessage 请求人类决策，实现人机协同控制。
- **Meta 的“智能体式生成”**：Meta 发布的 **Muse Image** 和 **Muse Video** 模型，其技术亮点不仅是图像质量，更在于其**显式的智能体式生成循环**：在最终渲染前，模型会进行规划、网络搜索、工具使用、代码执行和自我优化。这种自我优化行为是在强化学习（RL）过程中**涌现**的，而非硬编码。其性能随测试时计算的增加而提升，Muse Image 在 Image Arena 上迅速升至第二位。
- **解决特定失败模式**：**Liquid AI 的 Antidoom** 方法是一个典型案例，它专注于解决小型推理模型陷入“doom loop”（重复输出 token 直到上下文耗尽）的问题。其方法 **FTPO（最终 Token 偏好优化）** 通过重新标记触发循环的 token 并将概率重新分配给其他选项，将特定模型的循环率从 10.2% 降至 1.4%，另一个模型从 22.9% 降至 1%。这体现了业界从单纯扩大参数规模转向消除特定故障模式的趋势。
- **可解释性领域的“J-Space”争论**：Anthropic 关于 J-space 的工作主导了讨论，但也引发了尖锐批评。批评者认为，其发现的向量在很大程度上是因 **Jacobian 透镜**的定义方式而人为构建的。更有价值的技术发现是**跨模型结构**：@eliebakouch 计算了 38 个开放模型的 J-lens 几何相似性（CKA），发现其层/深度组织结构具有惊人的普遍性，甚至跨越了 Llama 和 OLMo 等不相关的模型家族。
- **基准测试揭示的差距**：**Harvey LAB-AA** 法律智能体基准测试显示，即便模型能满足许多单项标准，仍无法产出可接受的端到端成果。表现最好的 Claude Fable 5 在 120 个私法任务中的全部通过率也仅为 14.2%，而成本仅为其约 6% 的 GLM-5.2 也达到了 7.5% 的通过率。

### Implications for AI, investing, products, or research
- **对 AI 研究与产品**：框架工程正从“技巧”演变为一门独立的工程学科。未来的 AI 系统能力将越来越多地由模型权重和其外部框架共同定义。研究重点正从训练更强的模型，转向设计能更有效引导和放大模型能力的框架，包括元框架和自动化研究框架。
- **对投资**：框架层（Harness Layer）正在成为价值创造的新焦点。投资机会可能从基础模型本身，扩展到能够构建、管理和优化这些框架的工具和平台。Thinky、LangChain 等公司的动向，以及 Google、Anthropic 将框架深度整合进产品的行为，都表明这是一个关键的战略控制点。
- **对产品设计**：产品体验正从“与模型聊天”转向“与完成任务的后台智能体协作”。Claude Cowork 的推出是这一转变的明确信号。产品设计需要考虑如何让用户设定目标、监控进度、并在必要时进行干预，而不仅仅是进行对话。
- **对技术路径**：Lilian Weng 的观点为 RSI 的实现路径提供了重要参考。与其追求模型内部的自我修改（这充满风险且困难），不如专注于构建可迭代、可优化的外部框架，这是一条更务实、更可控的路径。同时，像 Antidoom 这样的工作表明，针对特定失败模式的“外科手术式”优化，是提升系统可靠性的高效方法。
