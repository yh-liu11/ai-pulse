## Summary

AI 基础设施正经历一场根本性的范式转移：从为人类开发者设计的“开发者体验”转向为自主编程的 AI 代理设计的“代理体验”。Modal 的 CTO Akshat Bubna 指出，传统云基础设施（尤其是 Kubernetes）是为人类设计的，人类可以阅读文档、理解 YAML 配置、查看仪表盘来填补上下文空白。但 AI 代理没有这种奢侈。代理需要一个能编写代码、运行、检查输出、更改环境、调试失败并快速重试的紧密反馈循环。因此，基础设施必须变得更程序化、更自服务、更可观测，以适应代理的操作方式。

**核心论点**

1.  **Kubernetes 不适合突发性 AI 工作负载**：Kubernetes 是为缓慢扩缩的 Web 服务设计的，无法应对 AI 推理和训练中常见的、需要快速大规模扩缩 GPU 的突发性计算需求。Modal 的起源正是为了构建一个更好的运行时，解决 Kubernetes 在开发者体验和弹性上的不足。
2.  **从“开发者体验”到“代理体验”的转变**：Modal 已将其 SDK 团队的思维模式从服务人类开发者转向服务 AI 代理。其核心优势在于，通过装饰器将基础设施需求与代码共同定位，代理只需修改少量代码即可获得自服务运行时，这比让代理阅读数百个 Kubernetes 文件并编写 YAML 要高效得多。
3.  **可观测性比阅读代码更重要**：当代理编写代码时，人类不再直接审查每一行代码。因此，强大的仪表盘和 CLI 工具变得至关重要，代理可以用它们进行自我调查，而人类则需要通过这些工具来解释正在发生的事情并做出判断。
4.  **沙箱是代理时代的核心原语**：代理需要隔离的环境来安全地执行代码、迭代和自我修正。Modal 早在 2023 年 5 月就构建了沙箱功能，并预见到代理循环的需求。在强化学习等场景中，一次 rollout 可能需要多达 10 万个沙箱，这对基础设施的弹性提出了极高要求。
5.  **弹性推理与 GPU 快照**：Modal 最大的用例是为自定义模型提供弹性推理服务。其核心竞争力在于自动扩缩容，并通过 GPU 快照技术将模型状态（如 `torch.compile` 后的模型）保存下来，使冷启动速度大幅加快，从而应对高度不可预测的流量。
6.  **超越原始 GPU 租赁的推理性能**：Modal 通过自研技术（如开源的 DeFlash 投机解码）达到前沿模型的推理性能，并推出 Auto Endpoints 等功能，让部署优化推理变得更容易，从而在 vLLM、SGLang 等开源框架之上提供额外价值。
7.  **“超级云”策略与网络创新**：Modal 不拥有自己的数据中心，而是在全球 17 个云提供商之上构建了一个容量池，并开发了自己的可靠性层。在网络方面，它提供了私有 IPv6 覆盖网络、RDMA 支持的无服务器多节点分布式训练，以及支持 sidecar 的网络化沙箱，以满足复杂的代理工作负载需求。
8.  **生产级代理需要硬性护栏**：对于生产环境中的代理，仅靠 LLM 介导的权限控制是不够的，必须在沙箱级别设置硬性边界，以防止数据泄露等安全问题。Modal 的定位是成为专业的沙箱提供商，为需要更多控制权的生产级代理提供支持。

**值得展开的细节**

*   **投机解码**：Modal 开源的 DeFlash 是一种基于块的投机解码技术。它使用一个小型“草案模型”预测多个 token，再由大型模型进行批量验证。由于批量验证更高效地利用了 GPU 的计算能力而非受限于内存带宽，因此能显著提升推理速度。
*   **代理生命周期与后台代理**：Modal 正在围绕模型生命周期和代理生命周期构建平台，覆盖从数据准备、训练到推理的全过程。Ramp Inspect 是一个成功的后台代理案例，它利用 Modal 的快照和快速扩缩容能力，实现了反应灵敏的后台运行体验。
*   **推理拐点**：AI 工作负载中 GPU 与 CPU 的比例正从过去的 8:1 向 1:1 转变。因为代理在执行任务时会频繁调用 CPU 密集型操作，导致计算瓶颈在 GPU 和 CPU 之间来回切换，这要求基础设施能够将两者紧密地协同定位。
*   **非 LLM 的 AI 工作负载**：除了大语言模型，Modal 还服务于音频、视频、机器人、计算生物学等领域的公司。这些公司训练自己的模型，但使用 Modal 进行部署和弹性扩缩，这表明其平台具有广泛的适用性。
*   **CI/CD 与编码代理**：Modal 看好 CI 市场的潜力，认为编码代理将运行更多的 CI 任务。通过内存快照和恢复等原语，可以显著减少 CI 流程中准备工件和依赖项所浪费的时间。
*   **Modal Bench**：为了衡量和改进代理体验，Modal 构建了一个基准测试，用于发现代理在哪些方面存在不足，并据此增加产品功能或 CLI 工具，将代理的“幻觉”需求转化为实际的产品反馈。

**对 AI、投资、产品和研究的影响**

*   **对 AI 基础设施**：未来的云平台必须从设计之初就考虑被 AI 代理操作，而不仅仅是被人使用。这意味着 API 需要更简洁、可编程性更强，并且需要提供丰富的可观测性接口。
*   **对投资**：Modal 的 3.55 亿美元 C 轮融资和其“代理云”定位，表明市场正在重金押注 AI 原生基础设施层。能够解决 AI 工作负载（尤其是代理）独特需求（如突发性、沙箱化、弹性推理）的公司具有巨大的增长潜力。
*   **对产品**：任何构建 AI 代理或相关工具的公司，都应关注底层基础设施的“代理体验”。选择或构建能让代理更快、更可靠地迭代和部署的基础设施，可能成为产品核心竞争力的来源。
*   **对研究**：基础设施的演进与模型能力的提升是相辅相成的。更好的沙箱和弹性计算环境，为强化学习、代理循环、持续学习等前沿研究提供了实验基础。同时，像投机解码这样的系统层面优化，对于提升模型推理效率至关重要。
